# 新加坡小学数学辅导 App 需求文档 (2026 更新版)

## 1. 项目概述

### 1.1 背景
新加坡教育部（MOE）于 2026 年正式执行新版 PSLE（小学离校考试）数学大纲。本项目利用 **Java 25 (LTS)** 的高性能并发特性与 **Spring AI 2.x** 的代理工作流（Agentic Workflow），打造一款面向 P1-P6 学生家长的 AI 智能辅导应用。重点攻克 2026 新大纲中的**代数（Algebra）**和**比与平均数（Ratio & Average）**等高分难点。

### 1.2 目标用户
- **主要用户**：P1-P6 学生的家长（辅导者角色），尤其是需要应对 2026 新大纲挑战的家长。
- **核心痛点**：家长难以理解新加坡独特的 CPA 教学法和 Model Method，且 2026 新增的代数入门对传统辅导方式提出了新要求。

---

## 2. 2026 新版 PSLE 数学知识树

### 2.1 核心变化（2026 关键点）
- **新增：代数入门 (Algebra)**：P6 必考，强调字母表示未知数及简单方程。
- **移除：速率 (Speed)**：已从核心考点中移除。
- **强化：Ratio & Average**：逻辑复杂度提升，成为区分优等生的关键。
- **提前：Nets & Pie Charts**：下沉至 P4 学习。

---

## 3. 功能需求

### 3.1 智能 Agent 解题器 (P0 - MVP)
- **文字输入**：家长手动输入或粘贴数学题目。
- **Agent 逻辑链**：
    1. **Planner Agent**：解析题目意图，拆解知识点与解题步骤。
    2. **CPA Designer Agent**：自动生成可视化 Bar Model 描述（JSON/SVG）。
    3. **Persona Agent**：生成针对家长的"教学导引"和针对孩子的"趣味脚本"。
- **流式响应**：基于 SSE 的即时反馈，首字延迟 < 500ms。

### 3.2 2026 知识点图谱 (P1)
- 覆盖全级次知识树，以 2026 大纲为基准动态更新。
- 标记"高频考点"与"新增考点"（如代数）。

### 3.3 OCR 拍照识题 (P2)
- 支持手机拍照上传题目图片，OCR 识别后进入解题 Agent 链。

### 3.4 学生档案与进度追踪 (P2)
- 家长账号下管理多个学生档案。
- 错题本、知识点掌握度评估。

---

## 4. 非功能需求 (2026 标准)

- **AI 模型**：
  - 生产环境：DeepSeek-R1（推理增强）或 Gemini 2.0 Pro。
  - 本地开发：Ollama + qwen3.5 （零 API 费用，离线可用）。
- **性能**：利用 Java 25 Virtual Threads + Structured Concurrency 处理高并发 Agent 请求。
- **隐私**：符合新加坡 PDPA，敏感数据加密，API 鉴权（JWT）。

---

## 5. MVP 路线图 (4 周冲刺)

| 周次 | 阶段 | 核心目标 | 交付物 |
| :--- | :--- | :--- | :--- |
| **Week 1** | **核心 Agent 搭建** | Java 25 + Spring AI 2.x 环境，集成 Ollama 本地模型，实现解题 Agent 链。 | 可运行的后端 API（/api/v1/solve/stream）。 |
| **Week 2** | **RAG 知识库** | PostgreSQL + pgvector 初始化，导入新加坡历年真题向量数据。 | 具备 PSLE 标准匹配能力的增强型解题器。 |
| **Week 3** | **Web 前端** | Compose Multiplatform (Wasm) 开发 Web 版原型。 | 可在线交互的解题工具 Web 端。 |
| **Week 4** | **闭环验证** | 端到端测试、Prompt 微调、部署上线。 | 内测版本 + 首批家长反馈。 |
